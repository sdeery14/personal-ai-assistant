# Data Model: Unified Eval Navigation

## Entities

### AgentVersion (MLflow LoggedModel)

Stored in MLflow's tracking server, not PostgreSQL. Each represents a unique git commit that ran evaluations.

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| model_id | string | `LoggedModel.model_id` | Primary identifier (e.g., "m-074689...") |
| name | string | `LoggedModel.name` | Auto-generated by MLflow git versioning |
| git_branch | string | `tags["mlflow.source.git.branch"]` | e.g., "main", "feature-branch" |
| git_commit | string | `tags["mlflow.source.git.commit"]` | Full SHA |
| git_dirty | boolean | `tags["mlflow.source.git.dirty"]` | Whether uncommitted changes existed |
| git_diff | string | `tags["mlflow.source.git.diff"]` | Uncommitted diffs (empty if clean) |
| git_repo_url | string | `tags["mlflow.source.git.repoURL"]` | Remote URL |
| creation_timestamp | integer | `LoggedModel.creation_timestamp` | Millis since epoch |
| experiment_id | string | `LoggedModel.experiment_id` | MLflow experiment this model belongs to |
| aggregate_quality | float | Computed | Average universal quality (1-5) across experiment types |

### AgentVersionDetail (Computed, not stored)

Assembled from multiple MLflow queries when viewing a single agent version.

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| model_id | string | LoggedModel | Primary identifier |
| git_metadata | object | LoggedModel tags | branch, commit, dirty, diff, repo_url |
| experiment_results | list | Aggregated from traces | Per-experiment pass rate, quality, run count |
| total_runs | integer | Count of linked runs | Via traces → source runs |
| total_traces | integer | Count of linked traces | `search_traces(model_id=...)` |

### Existing Entities (Unchanged)

These entities already exist from Features 014/015 and are reused without modification:

- **Experiment** (MLflow Experiment): experiment_id, name, eval_type, run_count, last_run
- **Run** (MLflow Run): run_id, experiment_id, params, metrics, start_time, status
- **Trace** (MLflow Trace): trace_id, run_id, inputs, outputs, assessments, spans
- **Dataset** (Golden dataset JSON files): name, description, version, cases

## Relationships

```
AgentVersion (LoggedModel)
    │
    ├── 1:N → Traces (linked via model_id)
    │           │
    │           └── N:1 → Run (each trace belongs to a run)
    │                       │
    │                       └── N:1 → Experiment
    │
    └── computed → ExperimentResults (aggregated per experiment type)

Experiment
    └── 1:N → Runs
                └── 1:N → Traces
                            └── 1:N → Assessments
```

## Identity & Uniqueness

- **AgentVersion**: Unique by `model_id`. MLflow's git versioning ensures one LoggedModel per unique (branch + commit + dirty state) combination. Same clean commit reuses existing model; dirty commits with different diffs create separate models.
- **Experiment, Run, Trace**: Identity unchanged from Features 014/015.

## No New Database Tables

All data lives in MLflow's tracking server. No PostgreSQL migrations needed.
